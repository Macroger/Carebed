# Class Sheet: ISensor

**Namespace:** Carebed.Models.Sensors

---

## Purpose
Defines the minimal contract for a sensor in the Carebed system. Implementations provide snapshots of sensor data and support lifecycle management (start/stop).

---

## Interface Definition
```
public interface ISensor
```

---

## Properties
- `string SensorID { get; init; }`
  - Logical source/id for the sensor (e.g., "tempSensor1", "RoomA_EEG").
  - Init-only property.

- `SensorTypes SensorType { get; init; }`
  - The type of sensor (e.g., Temperature, HeartRate, EEG).
  - Init-only property for classification and filtering.

---

## Methods
- `SensorData ReadData()`
  - Reads a single snapshot of sensor data.
  - Called by the SensorManager every poll cycle.
  - Returns a `SensorData` payload representing the current reading.

- `void Start()`
  - Starts periodic sampling if the sensor implements its own timer (optional).
  - Called by SensorManager when it is started.

- `void Stop()`
  - Stops periodic sampling (optional).
  - Called by SensorManager when it is stopped.

---

## Events
- `event Action<SensorStates> OnStateChanged`
  - Triggered when the sensor transitions to a new state.
  - Useful for emitting status messages or updating the UI.

---

## Notes
- Used by SensorManager to poll and manage sensors.
- Returns strongly-typed `SensorData` objects for event bus communication.
- The `OnStateChanged` event allows external listeners to react to sensor state transitions.
- Properties `SensorID` and `SensorType` are init-only to ensure immutability after creation.
