# Class Sheet: LoggingManager

**Namespace:** Carebed.Managers

---

## Purpose
Manages centralized logging of all event bus messages to file storage. Subscribes to global event bus messages, routes them to a file logging service, and handles logging configuration commands. Provides a unified logging hub for system-wide diagnostics and audit trails.

  - Processing Steps:

- `void Stop()`
  - Stops the logging service and unsubscribes from event bus messages.
  - Processing Steps:

	2. If subscribed: unsubscribes from global messages and logger commands.

	4. Sets `_isLoggerStarted` and `_isSubscribed` flags to false.

- `void Dispose()`
  - Disposes the LoggingManager and releases all resources.
  - Processing Steps:

## Usage Example
```csharp
// Create the logging manager
string logDir = "Logs";
string logFileName = "application.log";
var loggingService = new SimpleFileLogger(Path.Combine(logDir, logFileName));
var eventBus = new BasicEventBus();

var loggingManager = new LoggingManager(logDir, logFileName, loggingService, eventBus);

// Start logging
loggingManager.Start();

// Now all event bus messages are logged to file
// ... application runs ...

// Change log file location at runtime (while stopped)
loggingManager.Stop();
loggingManager.UpdateLogLocation("NewLogs", "new_application.log");
loggingManager.Start();

// Request current log file path via command
var commandMessage = new LoggerCommandMessage
{
	Command = LoggerCommands.GetLogFilePath
};
var envelope = new MessageEnvelope<LoggerCommandMessage>(
	commandMessage,
	MessageOrigins.UIComponent,
	MessageTypes.System
);
await eventBus.PublishAsync(envelope);
// LoggingManager publishes LoggerCommandAckMessage with file path in metadata

// Stop logging
loggingManager.Stop();
loggingManager.Dispose();
```

## Supported Commands

### LoggerCommands.Start
- Starts the logging service if not already started.
- Publishes positive ACK if successful, negative ACK if already running.

### LoggerCommands.Stop
- Stops the logging service if currently running.
- Publishes positive ACK if successful, negative ACK if not running.

### LoggerCommands.AdjustLogFilePath
- Updates the log directory and file name.
- Requires metadata with keys: "LogDirectory" and "FilePath".
- Can only be called when logger is stopped.
- Publishes ACK with success/failure result.

### LoggerCommands.GetLogFilePath
- Returns the current log file path.
- Publishes ACK with path information in metadata.
