# Class Sheet: SensorManager

**Namespace:** Carebed.Managers

---

## Purpose
Top-level manager that polls configured sensors on a single timer and publishes their readings to the application's event bus. Handles sensor lifecycle management, command routing, and state change notifications.

---

## Class Definition
```
public class SensorManager : IManager
```

---

## Constructors
- `SensorManager(IEventBus eventBus, List<AbstractSensor> sensorList, double intervalMilliseconds = 1000)`
  - Initializes a new instance of the `SensorManager` class with an event bus and a collection of sensors.
  - Parameters:
    - `eventBus`: The event bus used to publish sensor data (cannot be null).
    - `sensorList`: The collection of sensors to be managed and polled (cannot be null or empty).
    - `intervalMilliseconds`: The interval in milliseconds at which the sensors are polled (defaults to 1000 ms).
  - Throws `ArgumentNullException` if `eventBus` or `sensorList` is null.

## Public Methods

### Timer and Polling Control
- `void Start()`
  - Starts the timer and all managed sensors.

- `void Stop()`
  - Stops the timer and all managed sensors.

- `void StartTimer()`
  - Starts the internal polling timer.

- `void StopTimer()`
  - Stops the internal polling timer.

### Sensor-Specific Control
- `void StartSensor(string sensorId)`
  - Starts a specific sensor by its ID.

- `void StopSensor(string sensorId)`
  - Stops a specific sensor by its ID.

### Polling and Configuration
- `async Task PollOnceAsync()`
  - Executes a single polling operation, reading data from all sensors and publishing results to the event bus.
  - Ensures thread safety by preventing overlapping polls.
  - Handles errors gracefully and continues polling other sensors if one fails.

- `bool AdjustPollingRate(double intervalSeconds)`
  - Allows adjustment of the polling interval at runtime.
  - Parameters: `intervalSeconds` (must be between 0 and 60 seconds)
  - Returns `true` if adjustment was successful, `false` otherwise.

- `void HandleStateChanged(AbstractSensor sensor, SensorStates state)`
  - Handles sensor state change events.

## Usage Example
```csharp
var eventBus = new BasicEventBus();
var sensors = new List<AbstractSensor> 
{
    new TemperatureSensor { SensorID = "temp1", SensorType = SensorTypes.Temperature },
    new HeartRateSensor { SensorID = "hr1", SensorType = SensorTypes.HeartRate }
};

var manager = new SensorManager(eventBus, sensors, intervalMilliseconds: 1000);
manager.Start();

// Later, adjust polling rate
manager.AdjustPollingRate(2.0); // Poll every 2 seconds

// Stop when done
manager.Stop();
```

---

## Notes
- Uses a timer to poll sensors at a configurable interval.
- Publishes sensor data as `MessageEnvelope<SensorData>` to the event bus.
- Ensures thread safety for polling operations using interlocked exchange.
- Subscribes to sensor command messages and handles them asynchronously.
- Monitors sensor state changes and publishes corresponding status or error messages.
- Emits sensor inventory message on initialization to inform other modules of available sensors.
- Polling rate can be adjusted at runtime between 0 and 60 seconds.
