# Class Sheet: MessageTypes

**Namespace:** Carebed.Infrastructure.Enums

## Purpose
Enumerates the high-level categories or tags for messages in the Carebed system. Useful for routing, prioritization, filtering, and UI display.

## Values
- `Undefined = 0`
  - Default value for unspecified or unknown message types.

- `SensorData`
  - Sensor reading or telemetry data message.

- `Alert`
  - Alert message triggered by the system.

- `AlertClear`
  - Message to clear an active alert.

- `AlertAction`
  - Action taken in response to an alert.

- `AlertClearAck`
  - Acknowledgment that an alert has been cleared.

- `ActuatorCommand`
  - Command message sent to an actuator.

- `ActuatorCommandAck`
  - Acknowledgment that an actuator command was received.

- `ActuatorStatus`
  - Status message from an actuator.

- `ActuatorError`
  - Error message from an actuator.

- `ActuatorTelemetry`

- `ActuatorInventory`
  - Inventory or discovery message for available actuators.

- `SensorCommand`
  - Command message sent to a sensor.

- `SensorCommandAck`
  - Acknowledgment that a sensor command was received.

- `SensorInventory`
  - Inventory or discovery message for available sensors.

- `SensorError`
  - Error message from a sensor.

- `SensorStatus`
  - Status message from a sensor.

- `System`
  - System-level message (initialization, shutdown, etc.).

- `LoggerCommandResponse`
  - Response from logging commands or operations.
