# Class Sheet: AbstractEventBus

**Namespace:** Carebed.Infrastructure.EventBus

---

## Purpose
Provides an abstract base implementation of `IEventBus` that centralizes subscriber management logic and lifecycle operations. This class handles thread-safe subscription and unsubscription of message handlers while requiring subclasses to implement the actual message dispatch strategy via `PublishAsync<TPayload>()`.

## Thread Safety
- **Lock Protection**: All access to `_subscribers` and `_globalHandlers` is protected by `_lock`.
- **Safe Subscriptions**: Multiple threads can safely call `Subscribe` and `Unsubscribe` concurrently.
- **Safe Publishing**: Multiple threads can safely call `PublishAsync` concurrently.
- **Handler Copy**: `NotifyAll` creates a thread-safe copy of the global handlers list before iterating to prevent modification during iteration.

## Usage Example
```csharp
// AbstractEventBus is abstract; use a concrete subclass like BasicEventBus
IEventBus eventBus = new BasicEventBus();
eventBus.Initialize();

// Subscribe to a specific message type
Action<MessageEnvelope<SensorData>> sensorHandler = (envelope) =>
{
	Console.WriteLine($"Sensor data: {envelope.Payload.Value}");
};
eventBus.Subscribe(sensorHandler);

// Subscribe to all messages
Action<IMessageEnvelope> globalHandler = (envelope) =>
{
	Console.WriteLine($"Message: {envelope.MessageType}");
};
eventBus.SubscribeToGlobalMessages(globalHandler);

// Publish messages
var sensorData = new SensorData { Value = 98.6, Source = "Temp1", ... };
var envelope = new MessageEnvelope<SensorData>(
	sensorData, 
	MessageOrigins.SensorManager, 
	MessageTypes.SensorData
);
await eventBus.PublishAsync(envelope);

// Unsubscribe and shutdown
eventBus.Unsubscribe(sensorHandler);
eventBus.UnsubscribeFromGlobalMessages(globalHandler);
eventBus.Shutdown();
```

## Design Patterns
- **Template Method Pattern**: AbstractEventBus provides the subscription/unsubscription template, while subclasses implement `PublishAsync` for custom dispatch behavior.
- **Observer Pattern**: Handlers are observers that react to published events.
- **Strategy Pattern**: Different event bus implementations (BasicEventBus, advanced queued variants) can implement different dispatch strategies.

## Notes
- **Centralized Management**: All subscription logic is centralized in AbstractEventBus, reducing code duplication in subclasses.
- **Type Safety**: Generic constraints ensure handlers and messages are properly typed at compile time.
- **Thread-Safe**: All public operations are thread-safe by design using lock protection.
- **Handler Resolution**: `GetHandlersFor<TPayload>` supports type assignment compatibility, allowing handlers subscribed to base types to receive derived type messages.
- **Protected Access**: Subclasses can access `_subscribers` and `_lock` directly for advanced dispatch strategies.
- **Foundation for Subclasses**: BasicEventBus and other implementations extend AbstractEventBus to provide concrete dispatch strategies.
